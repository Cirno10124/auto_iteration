# 架构说明（auto_iteration）

## 1. 目标

`auto_iteration` 提供一个可循环执行的语音模型迭代流水线：采集/切分 -> 自动标注 -> 清单构建 -> 训练 -> 评估 -> 转换。

## 2. 目录与模块职责

- `orchestrator.py`：兼容入口（薄封装）
- `orchestrator_core/cli.py`：CLI 解析、配置加载、校验、日志初始化
- `orchestrator_core/config_loader.py`：JSONC 配置读取与 `--override` 应用
- `orchestrator_core/config_validation.py`：配置必填项/类型/范围校验
- `orchestrator_core/context.py`：speaker 维度上下文构建
- `orchestrator_core/pipeline.py`：主流程编排与异常策略
- `orchestrator_core/step_runner.py`：统一子进程步骤执行与错误日志
- `logging_utils.py`：统一日志格式与上下文字段注入
- `scripts/`：音频切分、标注、清单、训练、评估、转换等执行脚本

## 3. 主流程（高层）

1. 读取配置（`--config` 或 `--env`）
2. 应用 `--override`
3. 配置校验失败则快速终止
4. 初始化统一日志上下文（`speaker/iter/step/model`）
5. 按 speaker、按轮次执行步骤
6. 根据异常策略执行重试/跳过/终止

## 4. 关键设计约束

- 保持命令兼容：`python orchestrator.py --config ...`
- 主 CI 运行轻量 pytest（`unit` / `integration_light`），不覆盖已剥离的说话人分离重型栈
- 训练与推理脚本保持子进程调用（便于替换 stub 与独立调试）

## 5. 配置分层

- `orchestrator_config.dev.json`：开发
- `orchestrator_config.test.json`：测试（含 stubs）
- `orchestrator_config.prod.json`：生产

敏感信息通过环境变量注入（例如 `HF_HUB_TOKEN`）。
