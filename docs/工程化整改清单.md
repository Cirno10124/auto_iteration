# 工程化整改清单（跟进版）

> 依据 `docs/工程化规范.txt` 形成的分阶段整改清单。  
> 使用方式：按优先级推进，完成后勾选对应项，并在“验收标准”中自检。

## 使用说明

- 建议按 `P0 -> P1 -> P2` 顺序执行
- 每一项尽量拆成独立 PR，降低回归风险
- 每次完成后更新状态（`[ ]` -> `[x]`）并补充备注

---

## P0（本周优先，先保证可稳定交付）

### 1) 修复 `.gitignore` 规则错误

- [ ] 将 `record_control\`、`speaker_separator_env\` 改为跨平台写法（如 `/`）
- [ ] 去掉 `.gitignore` 自忽略项，确保规则文件可追踪
- [ ] 复查常见产物忽略规则是否完整（日志、缓存、临时文件）

**验收标准**
- `git status` 不再出现异常忽略行为
- 检索/工具链不再报 `.gitignore` 解析错误

---

### 2) 补齐 CI 测试门禁

- [ ] 在 `.github/workflows/ci.yml` 中加入 `pytest`
- [ ] 确保 PR 与 `main` 分支都执行测试
- [ ] 失败即阻断合并

**验收标准**
- CI 不再只有 lint，至少包含 lint + test
- 引入回归后，CI 能自动失败

---

### 3) 建立覆盖率基线

- [ ] 接入 `pytest-cov`
- [ ] 先设一个可达门槛（建议 40%~50%）
- [ ] 对核心路径优先补测：`orchestrator.py`、`src/scripts/dataset_manager.py`、`src/scripts/labeler.py`

**验收标准**
- CI 输出覆盖率报告
- 覆盖率低于门槛时 CI 失败

---

### 4) 修正弱断言测试

- [ ] 将“仅打印统计”的测试补充为可重复断言
- [ ] 对外部依赖测试（模型/网络/大数据）增加 `mark` 分层
- [ ] CI 默认仅跑稳定集（快测）

**验收标准**
- 测试结果可自动判定（不是靠人工看日志）
- CI 耗时可控且稳定

---

## P1（1~2 周，提升可维护与可扩展）

### 5) 拆分 `orchestrator.py`，降低耦合

- [ ] 抽离配置处理模块（如 `config_loader.py`）
- [ ] 抽离流程步骤模块（如 `pipeline_steps.py`）
- [ ] 抽离运行控制模块（如 `runner.py`）
- [ ] 为拆分后的模块补齐单元测试

**验收标准**
- 单文件职责清晰，核心流程可按模块单测
- 新增/替换步骤时不需要大范围改动

---

### 6) 统一日志与追踪上下文

- [x] 核心流程移除 `print`，统一使用 `logging`
- [x] 统一日志字段（建议：step、speaker_id、iteration_id、model）
- [x] 错误日志补齐关键上下文（输入文件、参数、阶段）

**验收标准**
- 日志格式统一，能按关键字段检索
- 故障排查路径清晰

---

### 7) 收敛异常处理策略

- [x] 减少裸 `except Exception`
- [x] 明确区分可恢复错误与致命错误
- [x] 明确重试、跳过、终止策略并落日志

**验收标准**
- 不再出现无声失败（silent failure）
- 失败行为可预期且可追踪

---

### 8) 环境与配置分层

- [x] 划分 `dev/test/prod` 配置
- [x] 敏感信息改用环境变量注入（如 token）
- [x] 增加配置校验（必填项、类型、范围）

**验收标准**
- 本地/测试/生产配置边界清晰
- 错误配置可在启动阶段被拦截

---

## P2（持续优化，完善工程能力）

### 9) 容器化与一键部署能力

- [x] 增加基础 `Dockerfile`（优先 CPU 版）
- [x] 视情况补充 `docker-compose`（含外部依赖）
- [x] 提供一键启动与最小部署说明

**验收标准**
- 新机器可通过容器快速启动
- 运行环境差异显著降低

### 10) 依赖与安全扫描治理

- [x] Python 接入 `pip-audit`（或等价工具）

- [x] 设定依赖升级节奏（如每月一次）

**验收标准**
- CI 可产出漏洞扫描结果
- 高危漏洞有可追踪处置流程

---

### 11) 前后端质量门禁统一

- [x] 将 `sign_tools` 的 lint/test/build 纳入主 CI
- [x] 统一代码质量策略（Python + 前端）
- [x] 明确哪些检查是必过项

**验收标准**
- 子项目不游离在质量门禁之外
- 一次 CI 可覆盖主要工程风险

---

### 12) 协作文档补全

- [x] 补齐架构说明文档
- [x] 补齐部署手册
- [x] 补齐故障排查手册
- [x] 补齐测试说明（如何本地跑、如何定位失败）

**验收标准**
- 新同学可在短时间内完成本地跑通
- 常见问题有标准化排查路径
