# 测试说明（auto_iteration）

## 1. 测试分层

- `unit`：纯逻辑单测（无模型/网络依赖）
- `integration_light`：轻量集成（使用 `tests/stubs`）
- `heavy`：真实模型/网络/GPU/大数据相关（默认不进主 CI）

## 2. 本地执行命令

### 2.1 主 CI 等价（推荐）

```bash
pytest -m "not heavy" -q
```

### 2.2 带覆盖率

```bash
pytest -m "not heavy" -q --cov=. --cov-config=.coveragerc --cov-report=term-missing
```

### 2.3 全量（含 heavy）

```bash
pytest -q
```

## 3. 常见用例入口

- 中控回归：`tests/end_to_end_test.py`
- 配置与上下文：`tests/test_orchestrator_unit.py`
- 数据清单逻辑：`tests/test_dataset_manager_unit.py`
- 相似度基础数学：`tests/test_similarity_metrics_unit.py`

## 4. 失败定位方法

1. 使用 `-k` 聚焦：

```bash
pytest -k orchestrator -q
```

2. 单文件复现：

```bash
pytest tests/end_to_end_test.py -q
```

3. 如为 heavy 失败，确认：
- 模型下载权限（`HF_HUB_TOKEN`）
- 数据路径可访问
- GPU 可用
