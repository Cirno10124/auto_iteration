# 验收部署与测试清单（全流程跑通版）

本文用于把 `auto_iteration` 交给他人测试验收时，保证可以按步骤独立完成部署、执行并给出通过结论。

---

## 0. 使用说明

- 目标：先确保**全流程可跑通**，再看指标是否达标。
- 适用对象：首次接手项目的测试/验收人。
- 执行原则：每一步执行后立即记录结果（成功/失败、日志路径、截图或关键输出）。
- 可直接使用脚本快速执行：
  - PowerShell：`scripts/acceptance_smoke.ps1`
  - Bash：`scripts/acceptance_smoke.sh`

---

## 0.1 一键执行命令（PowerShell + Bash）

### PowerShell（Windows）

- 默认执行：
  - `powershell -ExecutionPolicy Bypass -File scripts/acceptance_smoke.ps1`
- 指定配置：
  - `powershell -ExecutionPolicy Bypass -File scripts/acceptance_smoke.ps1 -ConfigPath config_template/orchestrator_config.test.json`
- 跳过依赖安装或 pytest（可选）：
  - `powershell -ExecutionPolicy Bypass -File scripts/acceptance_smoke.ps1 -SkipPipInstall -SkipPytest`

### Bash（Linux/macOS）

- 赋予执行权限：
  - `chmod +x scripts/acceptance_smoke.sh`
- 默认执行：
  - `./scripts/acceptance_smoke.sh`
- 指定配置：
  - `./scripts/acceptance_smoke.sh config_template/orchestrator_config.test.json`
- 跳过依赖安装或 pytest（可选）：
  - `SKIP_PIP_INSTALL=1 SKIP_PYTEST=1 ./scripts/acceptance_smoke.sh`

## 1. 验收前准备（必须完成）

### 1.1 代码与版本确认

- [ ] 获取指定验收版本代码（分支、tag 或提交号）。
- [ ] 记录版本信息：`git rev-parse --short HEAD`
- [ ] 确认当前目录为项目根目录 `auto_iteration/`。

### 1.2 运行环境确认

- [ ] 操作系统可用（Windows/Linux 任一，建议与生产环境一致）。
- [ ] Python 3.11 可用：`python --version`
- [ ] 安装依赖：`pip install -r requirements.txt`
- [ ] 若执行训练路径，GPU 可用：`nvidia-smi`

### 1.3 配置与凭据

- [ ] 准备配置文件：优先使用 `config_template/orchestrator_config.test.json`（如需复制到根目录请记录）。
- [ ] 明确本次测试数据目录（输入音频、输出目录、日志目录）。

## 3. 全流程冒烟（先验证“能跑完”）

目标：不纠结指标，先确认链路完整执行。

### 3.1 冒烟执行

- [ ] 运行一次完整流程（建议单 speaker）：
  - `python orchestrator.py --config config_template/orchestrator_config.test.json --override iteration.once=true`
- [ ] 主进程无异常退出（退出码为 0）。
- [ ] 各阶段脚本均被触发：`split_audio` / `labeler` / `dataset_manager` / `train_lora` / `evaluator` / `converter(可选)`。

### 3.2 产物检查

- [ ] 分段结果目录有音频分片文件。
- [ ] 标注结果目录有文本或标注产物。
- [ ] 数据集清单文件生成（train/val/test manifests 或 CSV）。
- [ ] 训练产物生成（如 `best_model` 或模型目录）。
- [ ] 评估报告生成（包含 WER/CER 或等价指标）。
- [ ] 如开启转换，GGML 产物生成。

## 4. 标准验收测试（功能 + 稳定性）

### 4.1 功能验收项

- [ ] 配置加载：`--config`、`--env`、`--override` 均可用。
- [ ] 异常策略可用（步骤失败时行为符合预期：重试/跳过/终止）。
- [ ] 多 speaker 运行可用（如本次范围包含并行/多说话人）。

### 4.2 稳定性与可观测性

- [ ] 连续执行 2 次流程无崩溃（可用同一配置重复跑）。
- [ ] 日志可追踪完整链路，能定位失败步骤与原因。
- [ ] 失败时错误信息可读，包含命令、返回码或关键信息。

### 4.3 回归测试（修复缺陷后）

- [ ] 缺陷场景复测通过。
- [ ] 相邻关键链路未回归（至少复测冒烟链路一次）。

## 7. 常见问题排查

- 依赖安装失败：优先确认 Python 版本与网络代理配置。
- GPU 相关失败：先跑 `nvidia-smi`，再跑 `gpu_health_check.py`。
- 模型下载失败：确认 `HF_HUB_TOKEN` 是否已注入且权限正确。
- 流程中断：查看统一日志中 `step` 字段，定位具体失败脚本。
