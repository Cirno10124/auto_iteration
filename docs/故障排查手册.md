# 故障排查手册（auto_iteration）

## 1. 启动即失败

### 现象

- 报 `配置校验失败`

### 排查

1. 检查使用的配置文件（`--config` 或 `--env`）
2. 对照 `orchestrator_core/config_validation.py` 的必填项与类型
3. 重点看：
   - `paths.*` 是否为非空字符串
   - `labeling.device` 是否为整值（`-1` 或 `0/1/...`）
   - `iteration.annotation_ratio` 是否在 `[0,1]`

## 2. 标注/训练步骤失败

### 现象

- 日志出现 `步骤失败` 或 `步骤执行异常`

### 排查

1. 先看日志上下文：`speaker/iter/step/model`
2. 查看该步骤完整命令（`cmd=`）并手工执行
3. 若是外部依赖问题，优先检查：
   - 模型下载权限（`HF_HUB_TOKEN`）
   - 磁盘空间与目录权限
   - GPU 可用性（`src/scripts/gpu_health_check.py`）

## 3. GPU 不可用

### 现象

- `torch.cuda.is_available() == False`
- 容器内 `nvidia-smi` 失败

### 排查

1. 主机执行 `nvidia-smi`
2. 检查 NVIDIA Container Toolkit 安装
3. 运行：

```bash
docker compose -f deploy/docker/docker-compose.gpu.yml run --rm orchestrator-gpu python3 src/scripts/gpu_health_check.py
```

## 4. CI 失败

### 常见类型

- Python lint/test 失败
- `sign_tools` 前端门禁失败（lint/test/build）
- 安全扫描发现漏洞

### 排查建议

1. 本地先跑：
   - `pytest -m "not heavy" -q`
   - `flake8 . --max-line-length=140 --extend-ignore=E203`
2. 前端：
   - `cd sign_tools && npm ci && npm run lint && npm run test && npm run build`
3. 安全扫描：
   - `pip-audit -r requirements.txt`
   - `cd sign_tools && npm audit`
