# 音频标注工具 — 需求清单

> 用于「模型预生成伪标签 + 人工校对」流程，便于在 Figma 中设计组件并落地实现。

---

## 1. 目标与角色

| 项目 | 说明 |
|------|------|
| **核心目标** | 高效校对 ASR 模型输出的伪标签，产出高质量训练/验证/测试集，并支持分级标签（A/B/C）。 |
| **主要用户** | 标注员（听音频 + 改文本）；可选：审核员（抽检）、管理员（任务与配置）。 |
| **典型场景** | 单人/小团队、本地或内网部署、支持与中控/任务队列对接。 |

---

## 2. 功能需求清单

### 2.1 任务与队列

| ID | 需求 | 优先级 | 说明 / Figma 可设计 |
|----|------|--------|----------------------|
| F1.1 | 任务列表 / 队列视图 | P0 | 展示「待标注 / 已标注 / 已跳过」列表，支持按优先级、置信度、来源筛选。可设计：列表行、筛选栏、状态标签。 |
| F1.2 | 单条任务信息展示 | P0 | 展示：音频文件名、时长、模型置信度、伪标签文本、可选：来源批次/场景标签。可设计：卡片或行内信息布局。 |
| F1.3 | 标注优先级与排序 | P1 | 支持按「置信度低优先」「中等置信度优先」或「顺序」排序，便于先处理最值得标的样本。可设计：排序/筛选控件。 |
| F1.4 | 与中控/脚本对接 | P1 | 能读取「待标样本列表 + 伪标签 + 置信度」；标注结果可写回文件或 API，供中控组训练集。可设计：导入/导出状态提示。 |

### 2.2 音频播放与片段

| ID | 需求 | 优先级 | 说明 / Figma 可设计 |
|----|------|--------|----------------------|
| F2.1 | 音频播放控制 | P0 | 播放 / 暂停、进度条、当前时间与总时长、倍速（如 0.5x, 1x, 1.5x）。可设计：播放器条组件。 |
| F2.2 | 精确定位 | P0 | 点击进度条跳转；支持「后退 N 秒」「前进 N 秒」按钮（如 ±3s）。可设计：进度条 + 两侧快捷按钮。 |
| F2.3 | 片段/句子级视图（可选） | P2 | 若数据带时间戳，按句或按段展示，点击某句可跳转到对应时间。可设计：时间轴 + 分段块。 |
| F2.4 | 波形可视化（可选） | P2 | 简单波形条，辅助判断静音与说话边界。可设计：波形条组件。 |

### 2.3 文本编辑与伪标签

| ID | 需求 | 优先级 | 说明 / Figma 可设计 |
|----|------|--------|----------------------|
| F3.1 | 伪标签展示与编辑 | P0 | 主编辑区展示当前样本的「模型输出文本」，可直接编辑；支持多行、复制粘贴。可设计：文本框/编辑器区域。 |
| F3.2 | 与音频联动 | P1 | 编辑时便于反复听对应片段（通过播放控制配合），无需额外切换。可设计：播放器与编辑区同屏布局。 |
| F3.3 | 差异/高亮（可选） | P2 | 若支持「建议修改」或规则检测，对疑似错误处高亮，便于快速定位。可设计：高亮样式、提示条。 |
| F3.4 | 原始模型输出保留 | P1 | 可查看「未修改前的伪标签」对比，或一键恢复。可设计：对比视图或「恢复」按钮。 |

### 2.4 标注动作与效率

| ID | 需求 | 优先级 | 说明 / Figma 可设计 |
|----|------|--------|----------------------|
| F4.1 | 接受当前标签 | P0 | 一键「通过 / 接受」，将当前文本记为最终标签并进入下一条。可设计：主按钮样式、位置。 |
| F4.2 | 提交并下一跳 | P0 | 保存当前修改并自动跳到下一条（顺序或按优先级）。可设计：主 CTA 按钮。 |
| F4.3 | 跳过 / 暂不标 | P0 | 标记为「跳过」或「待复核」，不写回为最终标签，可稍后处理。可设计：次要按钮。 |
| F4.4 | 键盘快捷键 | P0 | 如：空格=播放/暂停，Enter=提交并下一条，S=跳过，Esc=取消编辑等；需在界面有简短提示。可设计：快捷键说明浮层或侧栏。 |
| F4.5 | 批量操作（可选） | P2 | 多选若干条后「批量接受」或「批量跳过」。可设计：多选 checkbox、批量操作栏。 |

### 2.5 标签分级与质检

| ID | 需求 | 优先级 | 说明 / Figma 可设计 |
|----|------|--------|----------------------|
| F5.1 | 标签等级选择 | P1 | 提交时可选：A（严格人工审核）、B（快速过目）、C（仅模型伪标签未改）。可设计：单选/下拉、标签 chip。 |
| F5.2 | 问题/拒标标记 | P1 | 可选标记：无法听清、非目标语言、噪音过大、需复核等，便于后续过滤或再标。可设计：标签或下拉。 |
| F5.3 | 审核/抽检视图（可选） | P2 | 审核员查看「已标样本」列表，抽检并修正或打回。可设计：只读+批注或简单编辑视图。 |

### 2.6 统计与进度

| ID | 需求 | 优先级 | 说明 / Figma 可设计 |
|----|------|--------|----------------------|
| F6.1 | 当前任务进度 | P0 | 如「第 12 / 100 条」「已标注 45，跳过 3」。可设计：进度文字、进度条。 |
| F6.2 | 会话/批次统计（可选） | P2 | 本批次标注数、平均耗时、各等级数量。可设计：统计卡片或侧栏。 |

### 2.7 配置与数据

| ID | 需求 | 优先级 | 说明 / Figma 可设计 |
|----|------|--------|----------------------|
| F7.1 | 数据导入方式 | P0 | 支持从目录/CSV/JSON 读取：音频路径、伪标签、置信度等。可设计：设置页、文件选择。 |
| F7.2 | 标注结果导出 | P0 | 导出为 CSV/JSON，含：音频 ID、最终文本、等级、是否跳过、问题标记。可设计：导出按钮、格式选项。 |
| F7.3 | 本地/轻量部署 | P1 | 可本地运行，数据不依赖外网；可选内网 API 与中控同步。可设计：无（偏实现）。 |

## 3. 界面结构建议（便于 Figma 分帧）

| 区域 | 内容 | 建议组件 |
|------|------|----------|
| **顶部** | 任务筛选/排序、进度、快捷键说明入口 | 导航栏、筛选下拉、进度条、图标按钮 |
| **左侧（可选）** | 任务队列列表，当前条高亮 | 列表、列表项、高亮态 |
| **中央主区** | 音频播放器 + 伪标签编辑框 | 播放器条、文本输入区、按钮组 |
| **右侧（可选）** | 当前样本元信息、等级/问题选择、统计 | 信息卡片、单选/标签、小统计 |
| **底部** | 主操作：接受、提交并下一条、跳过 | 主按钮、次按钮、键盘提示 |

可先做「单条任务」主界面 + 「任务列表」两帧，再扩展设置与统计。

## 4. 交互与键盘（建议统一为一页说明）

| 按键 | 动作 |
|------|------|
| 空格 | 播放 / 暂停 |
| Enter | 保存当前并进入下一条 |
| S | 跳过当前 |
| Ctrl+Z / Cmd+Z | 文本撤销（仅编辑区） |
| ← / → 或 J / L | 后退 / 前进 N 秒（如 3s） |
| Esc | 取消当前编辑 / 关闭弹窗 |
| 可选：1/2/3 | 快速选等级 A/B/C |

---

## 5. 非功能需求

| 项目 | 说明 |
|------|------|
| **性能** | 大列表（如 1000+ 条）下滚动与切换不卡顿；音频预加载或按需加载。 |
| **兼容** | 支持 Chrome/Edge 最新版；如需本地部署可考虑 Electron 或 Tauri。 |
| **数据安全** | 标注结果可本地保存、可加密或脱敏后上传；不硬编码敏感路径。 |
| **可访问** | 主要操作支持键盘完成；焦点顺序合理；必要时支持基础屏幕阅读。 |

---

## 6. 与中控/现有流程的对接点

- **输入**：中控或脚本产出「待标列表」：`audio_path, pseudo_label, confidence, [optional] batch_id`。
- **输出**：标注工具产出「已标结果」：`audio_id, final_text, label_level (A/B/C), skipped, issue_tag`。
- **可选**：中控按「置信度 + 抽样策略」生成带优先级的队列，标注工具按优先级排序展示。

---

## 7. 实现与设计顺序建议

1. **Figma**：先做「单条任务主界面」（播放器 + 编辑 + 主操作）+ 「快捷键说明」+ 「任务列表项」。
2. **开发**：先实现 F1.x, F2.1–F2.2, F3.1, F4.1–F4.4, F6.1, F7.1–F7.2，保证最小可用闭环。
3. **迭代**：再加入 F5.1–F5.2（分级/问题标记）、F1.3（优先级排序）、F3.4（对比/恢复）。
